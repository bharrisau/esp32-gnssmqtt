<objective>
Fix BUG-3/BUG-4: After OTA from old firmware (which never wrote mqtt_tls to NVS), the tls field
defaults to absent. Any code path that interprets absence as true causes mbedTLS to attempt a TLS
handshake against a plain-MQTT broker, failing silently.

Fix: `load_mqtt_config` must default tls to false when key absent. `save_credentials` must write
`mqtt_tls=0` and `config_ver=1` on every save so future firmware can detect schema version.

Purpose: Ensure MQTT connects after OTA from old firmware; establish NVS schema version convention.
Output: provisioning.rs with fixed default + save; mqtt_connect wired to use the tls flag.
</objective>
