<objective>
Implement FEAT-1: rework the GPIO9 boot button to support two hold thresholds.

Current behaviour: hold 3s → SoftAP mode. No feedback during hold.
New behaviour:
  - 0–3s held: no action
  - 3s reached (still held): LED flashes fast (ButtonHold pattern) — user sees warning
  - Released at 3–10s: SoftAP mode (existing behaviour, now with prior LED feedback)
  - 10s reached (still held): LED goes steady off (danger zone)
  - Released at 10s+: erase NVS partition + reboot (factory reset — credentials cleared, OTA slot unchanged)

Purpose: Allow field recovery from corrupt NVS without reflash hardware.
Output: led.rs with ButtonHold state; main.rs GPIO9 monitor with 3-phase state machine + led_state access.
</objective>
