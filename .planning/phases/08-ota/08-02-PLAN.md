**Public API (what Plan 03 will call):**
```rust
pub fn spawn_ota(
    mqtt_client: Arc<Mutex<EspMqttClient<'static>>>,
    device_id: String,
    ota_rx: std::sync::mpsc::Receiver<Vec<u8>>,
) -> anyhow::Result<()>
```

**Thread stack size:** 16384 bytes (HTTP client + SHA state + 4096-byte read buffer + OTA handle exceed default 8192).

**ota_task() implementation steps (in order):**

1. Derive topic strings once at thread start:
   - `status_topic = format!("gnss/{}/ota/status", device_id)`
   - `trigger_topic = format!("gnss/{}/ota/trigger", device_id)`

2. `for payload in &ota_rx` — blocks waiting for trigger bytes from the pump.

3. Parse the JSON trigger payload with minimal extraction (no serde). Extract `url` (required) and `sha256` (required — reject without it):
   ```rust
   fn extract_json_str<'a>(json: &'a str, key: &str) -> Option<&'a str> {
       let search = format!("\"{}\":\"", key);
       let start = json.find(&search)? + search.len();
       let end = json[start..].find('"')? + start;
       Some(&json[start..end])
   }
   ```
   If `url` or `sha256` is absent, publish `{"state":"failed","reason":"missing url or sha256"}` and continue to next iteration.

4. Publish `{"state":"downloading","progress":0}` to status_topic (QoS::AtMostOnce, retain=false).

5. Build HTTP client:
   ```rust
   let http_conf = HttpConfig { buffer_size: Some(4096), timeout: Some(Duration::from_secs(30)), ..Default::default() };
   let mut http = HttpClient::wrap(EspHttpConnection::new(&http_conf)?);
   let request = http.get(&url, &[])?;
   let mut response = request.submit()?;
   ```
   If HTTP setup fails, publish failed status with reason and continue.

   Check `response.status()` — if not 200, publish failed with `format!("HTTP {}", response.status())` and continue.

6. Create EspOta in a block to ensure it drops if initiate fails:
   ```rust
   let mut ota = match EspOta::new() {
       Ok(o) => o,
       Err(e) => { publish failed; continue; }
   };
   let mut update = match ota.initiate_update() {
       Ok(u) => u,
       Err(e) => { publish failed; continue; }
   };
   ```
   Note: `initiate_update()` blocks for 4-8 seconds during partition erase — this is expected. WDT timeout is extended to 30s in sdkconfig.

7. Streaming download + write loop:
   ```rust
   let mut hasher = Sha256::new();
   let mut buf = vec![0u8; 4096];
   let mut bytes_written: u64 = 0;
   let mut last_progress: u64 = 0;
   loop {
       let n = response.read(&mut buf)?;
       if n == 0 { break; }
       hasher.update(&buf[..n]);
       update.write(&buf[..n])?;
       bytes_written += n as u64;
       // Publish progress every 65536 bytes to avoid flooding broker
       if bytes_written - last_progress >= 65536 {
           last_progress = bytes_written;
           publish_status(&mqtt_client, &status_topic,
               &format!("{{\"state\":\"downloading\",\"progress\":{}}}", bytes_written));
       }
   }
   ```
   If read or write fails, publish failed status; `update` drops automatically calling abort. Continue to next trigger.

8. SHA-256 verification:
   ```rust
   let hash: [u8; 32] = hasher.finalize().into();
   let actual_hex: String = hash.iter().map(|b| format!("{:02x}", b)).collect();
   if actual_hex != sha256 {
       publish failed with format!("sha256 mismatch: expected {} got {}", sha256, actual_hex);
       // update drops here, calling esp_ota_abort automatically
       continue;
   }
   ```

9. Finalize: `update.complete()?` — if this fails, publish failed and continue (update already finalized; do not restart).

10. Publish `{"state":"complete"}` to status_topic.

11. Clear retained trigger: publish empty payload (`b""`) with retain=true to trigger_topic (QoS::AtLeastOnce).

12. Sleep 500ms to allow status MQTT message to be enqueued before restart.

13. `esp_idf_svc::hal::reset::restart()` — does not return.

**publish_status helper** (private fn, reuse across all status publishes):
```rust
fn publish_status(client: &Arc<Mutex<EspMqttClient<'static>>>, topic: &str, json: &str) {
    match client.lock() {
        Err(e) => log::warn!("OTA: status mutex poisoned: {:?}", e),
        Ok(mut c) => match c.enqueue(topic, QoS::AtMostOnce, false, json.as_bytes()) {
            Ok(_) => {}
            Err(e) => log::warn!("OTA: status enqueue failed: {:?}", e),
        },
    }
}
```

**Error handling pattern:** All errors in the download loop publish a failed status and `continue` to the next iteration — they do NOT panic or call restart(). The OTA thread keeps running and will accept another trigger.
