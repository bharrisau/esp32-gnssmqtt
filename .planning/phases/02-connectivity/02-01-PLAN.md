wifi_connect signature:
```rust
pub fn wifi_connect(
    modem: impl esp_idf_svc::hal::peripheral::Peripheral<P = esp_idf_svc::hal::modem::Modem> + 'static,
    sysloop: EspSystemEventLoop,
    nvs: EspDefaultNvsPartition,
) -> anyhow::Result<BlockingWifi<EspWifi<'static>>>
```

wifi_connect body:
1. Wrap EspWifi in BlockingWifi: `BlockingWifi::wrap(EspWifi::new(modem, sysloop.clone(), Some(nvs))?, sysloop)?`
2. Set configuration: `Configuration::Client(ClientConfiguration { ssid: crate::config::WIFI_SSID.try_into().unwrap(), password: crate::config::WIFI_PASS.try_into().unwrap(), auth_method: AuthMethod::WPA2Personal, ..Default::default() })`
3. Call sequence: `wifi.start()? → wifi.connect()? → wifi.wait_netif_up()?`
4. Return Ok(wifi)

wifi_supervisor signature:
```rust
pub fn wifi_supervisor(mut wifi: BlockingWifi<EspWifi<'static>>) -> !
```

wifi_supervisor body (runs forever in a dedicated thread):
- `backoff_secs: u64 = 1`
- Outer loop: sleep 5 seconds, then check `wifi.is_connected().unwrap_or(false)`
- If NOT connected: log warn with backoff_secs, sleep backoff_secs, then call `wifi.connect()` (NOT wifi.start() again — the driver is already started)
  - On Ok: call `wifi.wait_netif_up()`, log info "WiFi reconnected", reset backoff_secs to 1
  - On Err: log error, double backoff_secs, cap at 60: `backoff_secs = (backoff_secs * 2).min(60)`
- If connected: continue (do nothing this iteration)

Anti-patterns to avoid (from research):
- Do NOT call wifi.start() inside the reconnect loop — only call it once in wifi_connect
- Do NOT call wifi.connect() immediately after wifi.disconnect() — always add the sleep before reconnecting
- Do NOT use a flag for intentional disconnect in Phase 2 — we never call disconnect intentionally here

Add module-level doc comment: "WiFi connection and reconnect supervisor. Uses BlockingWifi<EspWifi> from esp-idf-svc."

The anyhow crate is available via esp-idf-svc's re-exports or add it as a dev convenience — check if anyhow is already in Cargo.toml. If not, use `esp_idf_svc::sys::EspError` as the error type and convert with `?` using the existing `From<EspError>` impls, OR add anyhow = "1" to Cargo.toml [dependencies] for cleaner error handling. The research examples use `anyhow::Result` — add anyhow if not present.
