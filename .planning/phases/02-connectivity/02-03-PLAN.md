<interfaces>
Hardware pin assignment (XIAO ESP32-C6 confirmed via Seeed documentation):
- UART1 RX = GPIO20 (receives data FROM UM980)
- UART1 TX = GPIO21 (sends data TO UM980)
- UART0 = USB Serial/JTAG CDC (the virtual COM port seen on the host PC)

UartDriver API (esp-idf-hal =0.45.2):
```rust
use esp_idf_svc::hal::uart::{UartDriver, config::Config};
use esp_idf_svc::hal::units::Hertz;

// Constructor: (uart_peripheral, tx_pin, rx_pin, cts_pin, rts_pin, config)
let driver = UartDriver::new(
    peripherals.uart1,
    peripherals.pins.gpio21,   // TX to UM980
    peripherals.pins.gpio20,   // RX from UM980
    Option::<esp_idf_svc::hal::gpio::AnyIOPin>::None,  // no CTS
    Option::<esp_idf_svc::hal::gpio::AnyIOPin>::None,  // no RTS
    &Config::new()
        .baudrate(Hertz(115_200))
        .rx_buffer_size(4096),  // matches config::UART_RX_BUF_SIZE
)?;

// Read: fills buf, returns bytes read; timeout in ticks (NON_BLOCK = 0, BLOCK = u32::MAX)
driver.read(&mut buf, NON_BLOCK)?;   // use esp_idf_svc::hal::delay::NON_BLOCK

// Write: returns bytes written
driver.write(&buf[..n])?;
```

USB CDC stdin/stdout (XIAO ESP32-C6 uses USB Serial/JTAG, not a separate USB-UART chip):
```rust
use std::io::{BufRead, BufReader, Write};
// Reading from USB input:
let stdin = std::io::stdin();
let mut reader = BufReader::new(stdin.lock());
reader.read_line(&mut line)?;   // blocks until newline
// Writing to USB output:
std::io::stdout().write_all(data)?;
std::io::stdout().flush()?;
```

Open question from research: Whether std::io::stdin() on XIAO ESP32-C6 maps to the USB JTAG CDC port or UART0 hardware GPIO16/17 is unverified. Implement using stdin/stdout (the standard ESP-IDF std mapping) — the human verification checkpoint in Plan 04 will confirm whether it works. If stdin() does not work, the bridge may need to use UartDriver on UART0 instead, but that is a deviation to handle in Plan 04's checkpoint.

Thread spawning with stack size (required to avoid FreeRTOS stack overflow):
```rust
std::thread::Builder::new()
    .stack_size(8192)
    .spawn(move || { ... })
    .unwrap();
```

**spawn_bridge** function:
```rust
pub fn spawn_bridge(
    uart1: impl Peripheral<P = impl Uart> + 'static,
    tx_pin: impl Peripheral<P = impl esp_idf_svc::hal::gpio::OutputPin> + 'static,
    rx_pin: impl Peripheral<P = impl esp_idf_svc::hal::gpio::InputPin> + 'static,
) -> anyhow::Result<()>
```

Body:
1. Create UartDriver:
   ```rust
   let um980 = UartDriver::new(
       uart1,
       tx_pin,
       rx_pin,
       Option::<AnyIOPin>::None,
       Option::<AnyIOPin>::None,
       &Config::new()
           .baudrate(Hertz(115_200))
           .rx_buffer_size(crate::config::UART_RX_BUF_SIZE as u32),
   )?;
   ```
2. Wrap in Arc for sharing: `let um980 = Arc::new(um980);`
3. Clone Arc for UM980→USB thread: `let um980_rx = Arc::clone(&um980);`

**Thread A — UM980 → USB (UM980 output echoed to serial console):**
```rust
std::thread::Builder::new()
    .stack_size(8192)
    .spawn(move || {
        let mut buf = [0u8; 256];
        loop {
            match um980_rx.read(&mut buf, NON_BLOCK) {
                Ok(n) if n > 0 => {
                    let _ = std::io::stdout().write_all(&buf[..n]);
                    let _ = std::io::stdout().flush();
                }
                _ => std::thread::sleep(std::time::Duration::from_millis(10)),
            }
        }
    })
    .unwrap();
```

**Thread B — USB → UM980 (USB serial input forwarded to UM980):**
```rust
std::thread::Builder::new()
    .stack_size(8192)
    .spawn(move || {
        let stdin = std::io::stdin();
        let mut reader = BufReader::new(stdin.lock());
        let mut line = String::new();
        loop {
            line.clear();
            match reader.read_line(&mut line) {
                Ok(0) => break,   // EOF
                Ok(_) => {
                    let _ = um980.write(line.as_bytes());
                }
                Err(e) => {
                    log::warn!("UART bridge stdin read error: {:?}", e);
                }
            }
        }
        log::error!("UART bridge USB->UM980 thread exited");
    })
    .unwrap();
```

Return `Ok(())`.

Note on UartDriver Send: If the compiler rejects moving `um980` into a thread (not Send), wrap in `Arc<Mutex<UartDriver>>` instead of plain `Arc`, and add mutex locks around read/write. Plain Arc works if UartDriver implements Send — check compiler output first.

Note on rx_buffer_size type: The Config::rx_buffer_size() method may accept usize or u32 depending on exact API version. Cast as needed: `.rx_buffer_size(crate::config::UART_RX_BUF_SIZE as u32)` or `as usize`.
