```rust
use esp_idf_svc::mqtt::client::{
    EspMqttClient, EspMqttConnection,
    LwtConfiguration, MqttClientConfiguration,
};
use embedded_svc::mqtt::client::{EventPayload, QoS};

// Constructor returns (client, connection) pair
let (client, connection) = EspMqttClient::new(&broker_url, &conf)?;

// Publish: topic, qos, retain, payload
client.publish(&topic, QoS::AtMostOnce, true, b"online")?;

// Subscribe: topic, qos
client.subscribe(&topic, QoS::AtLeastOnce)?;

// Connection pump: MUST run before any publish/subscribe
while let Ok(event) = connection.next() {
    match event.payload() {
        EventPayload::Connected(_) => { /* re-subscribe here */ }
        EventPayload::Disconnected => { /* log */ }
        EventPayload::Error(e) => { /* log */ }
        _ => {}
    }
}

// LwtConfiguration — topic &str MUST outlive conf
LwtConfiguration {
    topic: &lwt_topic,   // &str; declare lwt_topic before conf in same scope
    payload: b"offline",
    qos: QoS::AtLeastOnce,
    retain: true,
}
```

Thread safety note: Move EspMqttConnection into pump thread. Wrap EspMqttClient in Arc<Mutex<EspMqttClient<'static>>> for sharing across heartbeat and main threads.

**mqtt_connect** function:
```rust
pub fn mqtt_connect(
    device_id: &str,
) -> anyhow::Result<(Arc<Mutex<EspMqttClient<'static>>>, EspMqttConnection)>
```

Body:
1. Build broker URL: `format!("mqtt://{}:{}", crate::config::MQTT_HOST, crate::config::MQTT_PORT)`
2. Build LWT topic: `format!("gnss/{}/status", device_id)` — MUST be declared as a `let` binding BEFORE `conf` in the same scope (lifetime constraint — see pitfall 1 in research)
3. Build `MqttClientConfiguration`:
   - `client_id: Some(device_id)`
   - `username: if crate::config::MQTT_USER.is_empty() { None } else { Some(crate::config::MQTT_USER) }`
   - `password: if crate::config::MQTT_PASS.is_empty() { None } else { Some(crate::config::MQTT_PASS) }`
   - `lwt: Some(LwtConfiguration { topic: &lwt_topic, payload: b"offline", qos: QoS::AtLeastOnce, retain: true })`
   - `keep_alive_interval: Some(std::time::Duration::from_secs(60))`
   - `reconnect_timeout: Some(std::time::Duration::from_secs(5))`
   - `..Default::default()`
4. Call `EspMqttClient::new(&broker_url, &conf)?` — returns `(client, connection)`
5. Return `Ok((Arc::new(Mutex::new(client)), connection))`

**pump_mqtt_events** function:
```rust
pub fn pump_mqtt_events(
    mut connection: EspMqttConnection,
    client: Arc<Mutex<EspMqttClient<'static>>>,
    device_id: String,
) -> !
```

Body (runs forever — pump thread MUST start before any publish/subscribe):
- `while let Ok(event) = connection.next()` loop
- Match on `event.payload()`:
  - `EventPayload::Connected(_)` → log::info!("MQTT connected — re-subscribing"); then lock client and subscribe to `format!("gnss/{}/config", device_id)` at QoS::AtLeastOnce (ignore subscribe error with `let _ =`)
  - `EventPayload::Disconnected` → log::warn!("MQTT disconnected")
  - `EventPayload::Error(e)` → log::error!("MQTT error: {:?}", e)
  - `_ => {}` (ignore other events)
- After loop ends: `log::error!("MQTT pump exited — connection closed"); loop { std::thread::sleep(std::time::Duration::from_secs(60)); }` (prevent thread exit; connection is permanent)

**heartbeat_loop** function:
```rust
pub fn heartbeat_loop(
    client: Arc<Mutex<EspMqttClient<'static>>>,
    device_id: String,
) -> !
```

Body:
- Build topic once: `let topic = format!("gnss/{}/heartbeat", device_id)`
- Initial sleep of 5 seconds before first publish (give MQTT time to fully connect)
- Loop: sleep 30 seconds, then lock client and call `client.publish(&topic, QoS::AtMostOnce, true, b"online")` — log warn on error, ignore Ok. Use `if let Ok(mut c) = client.lock()` pattern (don't panic on poison)

Note: Do NOT use client.enqueue() — use client.publish() for heartbeats from a dedicated thread. The pump thread keeps the outbox moving.
