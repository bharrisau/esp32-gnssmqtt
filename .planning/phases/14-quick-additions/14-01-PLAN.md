<objective>
Enable ISO wall-clock timestamps in log output by initialising the SNTP client after WiFi connects and switching the ESP-IDF log timestamp source from RTOS ticks to system time.

Purpose: Operators reading serial monitor or MQTT log output will see real HH:MM:SS.mmm timestamps rather than ms-since-boot ticks, making log correlation across devices and sessions practical.
Output: sdkconfig.defaults updated with timestamp Kconfig; main.rs initialises EspSntp after wifi_connect and keeps the handle alive in main scope.
</objective>

<task type="auto">
  <name>Task 1: Add CONFIG_LOG_TIMESTAMP_SOURCE_SYSTEM to sdkconfig.defaults</name>
  <files>sdkconfig.defaults</files>
  <action>
Append the following block to the end of sdkconfig.defaults, after the existing CONFIG_ESP_TASK_WDT_PANIC=y line:

```
# Log timestamps: use system time (wall clock) instead of RTOS ticks.
# After SNTP syncs, log lines show HH:MM:SS.mmm instead of uptime ms.
# Requires EspSntp to be initialised at runtime (see main.rs Step 6.5).
CONFIG_LOG_TIMESTAMP_SOURCE_SYSTEM=y
```

This is a build-time Kconfig setting that switches EspLogger from `esp_log_timestamp()` (ms since boot) to `esp_log_system_timestamp()` (HH:MM:SS.mmm). The sdkconfig change invalidates the ESP-IDF CMake cache; the executor MUST run `cargo clean && cargo build --release` (not just `cargo build --release`) after this change to force CMake regeneration.

Do NOT add CONFIG_LOG_TIMESTAMP_SOURCE_RTOS or any conflicting setting.
