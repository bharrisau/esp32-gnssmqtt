<objective>
Create the `gnss-nvs` crate at `crates/gnss-nvs/` with a clean-room `NvsStore` trait and two backing implementations: the ESP-IDF NVS impl (wraps `EspNvs`) and a sequential-storage flash impl skeleton (compiles for no_std; hardware validation deferred).

Purpose: Establish the NVS abstraction layer so firmware can be ported from ESP-IDF NVS to sequential-storage without changing call sites. This is the foundational gap crate that enables the embassy port path.

Output: `crates/gnss-nvs/` with trait + two feature-gated impls, both `cargo check` passing.
</objective>

NvsStore trait (from RESEARCH.md Pattern 4):
```rust
pub trait NvsStore {
    type Error: core::fmt::Debug;
    fn get<T: serde::de::DeserializeOwned>(&self, namespace: &str, key: &str) -> Result<Option<T>, Self::Error>;
    fn set<T: serde::Serialize>(&mut self, namespace: &str, key: &str, value: &T) -> Result<(), Self::Error>;
    fn get_blob<'a>(&self, namespace: &str, key: &str, buf: &'a mut [u8]) -> Result<Option<&'a [u8]>, Self::Error>;
    fn set_blob(&mut self, namespace: &str, key: &str, data: &[u8]) -> Result<(), Self::Error>;
}
```
