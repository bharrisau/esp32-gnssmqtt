        Err(e) => { break; } // reconnect

    }
}

  <action>
    Create `gnss-server/src/mqtt.rs`:

    Define MqttMessage enum for internal channel passing (design at Claude's discretion):
    ```rust
    #[derive(Debug)]
    pub enum MqttMessage {
        Rtcm(bytes::Bytes),
        Nmea(bytes::Bytes),
        Heartbeat(bytes::Bytes),
    }
    ```

    Define `pub async fn mqtt_supervisor(config: Arc<ServerConfig>, msg_tx: tokio::sync::mpsc::Sender<MqttMessage>, state_tx: tokio::sync::watch::Sender<bool>)`:

    - Build `MqttOptions` from `config.mqtt` fields (broker, port, client_id, optional username/password)
    - Backoff array: `[1u64, 2, 5, 10, 30]`; `fail_count: usize = 0`
    - Outer loop: `AsyncClient::new(opts.clone(), 64)` — create fresh client+eventloop each reconnect cycle
    - Subscribe to `gnss/{device_id}/rtcm`, `gnss/{device_id}/nmea`, `gnss/{device_id}/heartbeat` at QoS::AtMostOnce immediately (they'll be sent when poll starts)
    - Inner poll loop: match EventLoop::poll().await:
      - ConnAck → `state_tx.send(true)`, `fail_count = 0`, re-subscribe (confirm subscriptions went through)
      - Publish → match topic suffix to MqttMessage variant, `msg_tx.try_send(msg)` (non-blocking; drop on full)
      - Err → `state_tx.send(false)`, log warn, break inner loop
      - _ → continue
    - After inner loop breaks: `let delay = backoff_steps[fail_count.min(4)]; fail_count += 1; tokio::time::sleep(Duration::from_secs(delay)).await`

    Note from RESEARCH.md Pitfall 5: subscribe BEFORE the poll loop starts will enqueue subscriptions in the channel; they are sent when the first poll() runs. No race condition.

    Add `#[cfg(test)]` module:
    - `test_backoff_sequence`: verify backoff_steps[0..5] == [1,2,5,10,30], backoff_steps[5.min(4)] == 30

    - `test_topic_to_variant`: given a topic string suffix, correct MqttMessage variant is constructed (pure function, no broker)
