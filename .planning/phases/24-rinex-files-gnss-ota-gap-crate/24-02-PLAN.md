**Step 1 — to_d19_12 formatter.**

Implement in rinex_writer.rs:

```rust
/// Format f64 as RINEX D19.12 (Fortran double-precision notation).
/// Output is always exactly 19 characters.
/// Examples: "  0.000000000000D+00", " 1.234567890123D-04"
pub fn to_d19_12(val: f64) -> String {
    if !val.is_finite() || val == 0.0 {
        return "  0.000000000000D+00".to_string();
    }
    // Rust format: {:19.12E} gives " 1.234567890123E-4" — need D and two-digit exponent with sign
    let s = format!("{:19.12E}", val);
    // Find 'E' separator
    if let Some(e_pos) = s.find('E') {
        let mantissa = &s[..e_pos];
        let exp: i32 = s[e_pos+1..].parse().unwrap_or(0);
        format!("{}D{:+03}", mantissa, exp)
    } else {
        "  0.000000000000D+00".to_string()
    }
}
```

Verify output width: format!("{:+03}", 4) = "+04", format!("{:+03}", -4) = "-04", format!("{:+03}", 14) = "+14". This satisfies the two-digit requirement.

**Step 2 — GPS week tracking.**

```rust
/// Compute current GPS week from UTC system clock.
/// GPS epoch: 1980-01-06 00:00:00 UTC.
pub fn current_gps_week() -> u32 {
    use chrono::TimeZone;
    let gps_epoch = Utc.with_ymd_and_hms(1980, 1, 6, 0, 0, 0).unwrap();
    let now = Utc::now();
    let days = (now - gps_epoch).num_days();
    (days / 7) as u32  // modulo 1024 not applied — use full week for UTC conversion
}

/// Convert GPS time-of-week (ms) + GPS week to UTC DateTime.
pub fn gps_tow_to_utc(gps_week: u32, tow_ms: u32) -> DateTime<Utc> {
    use chrono::{Duration, TimeZone};
    let gps_epoch = Utc.with_ymd_and_hms(1980, 1, 6, 0, 0, 0).unwrap();
    let total_ms = gps_week as i64 * 7 * 24 * 3600 * 1000 + tow_ms as i64;
    // GPS time is ahead of UTC by leap seconds; as of 2026, GPS-UTC = 18 seconds
    // Use 18 leap second correction (hardcoded; next leap second event unknown)
    gps_epoch + Duration::milliseconds(total_ms) - Duration::seconds(18)
}
```

Store gps_week in RinexNavWriter, initialized by caller with `current_gps_week()`.

**Step 4 — GPS navigation record writer.**

Per RESEARCH.md Pattern 4. Each GPS nav record = 8 lines × 80 chars.

Line 1 (PRN/EPOCH/SV CLK):
```
{prn:2} {yy:02} {mm:2} {dd:2} {hh:2} {min:2}{sec:5.1}{af0:19}{af1:19}{af2:19}
```

- PRN is satellite_id from Msg1019T

Lines 2-8 (BROADCAST ORBIT 1-7): each line starts with `   ` (3 spaces) then 4 × D19.12.

**Step 5 — GLONASS navigation record writer.**

Line 1 (SLOT/EPOCH/SV CLK):
```
{slot:2} {yy:02} {mm:2} {dd:2} {hh:2} {min:2}{sec:5.1}{clock_bias:19}{freq_drift:19}{message_frame_time:19}
```
- slot = satellite_id from Msg1020T

- freq_drift = gamma_n

Change run_decode_task to accept output_dir and station (from config) and initialize the RINEX writers:

```rust
async fn run_decode_task(
    mut msg_rx: mpsc::Receiver<mqtt::MqttMessage>,
    output_dir: String,
    station: String,
) {
    let gps_week = rinex_writer::current_gps_week();
    let mut epoch_buf = epoch::EpochBuffer::new();
    let mut obs_writer = rinex_writer::RinexObsWriter::new(&output_dir, station.clone(), gps_week);
    let mut nav_writer = rinex_writer::RinexNavWriter::new(&output_dir, station, gps_week);

    while let Some(msg) = msg_rx.recv().await {
        match msg {
            mqtt::MqttMessage::Rtcm(payload) => {
                let events = rtcm_decode::decode_rtcm_payload(&payload, &mut epoch_buf);
                for event in events {
                    match event {
                        observation::RtcmEvent::Epoch(group) => {
                            let epoch_utc = rinex_writer::gps_tow_to_utc(gps_week, group.epoch_ms);
                            if let Err(e) = obs_writer.write_group(&epoch_utc, &group) {
                                log::warn!("RINEX obs write error: {e}");
                            }
                        }
                        observation::RtcmEvent::Ephemeris(eph) => {
                            let epoch_utc = chrono::Utc::now();
                            if let Err(e) = nav_writer.write_ephemeris(&epoch_utc, &eph) {
                                log::warn!("RINEX nav write error: {e}");
                            }
                        }
                    }
                }
            }
            _ => {}  // Nmea and Heartbeat ignored by decode task
        }
    }
}
```
