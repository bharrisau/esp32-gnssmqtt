**# esp32-gnssmqtt**
One-sentence description: ESP32 firmware in Rust that streams GNSS data (NMEA + RTCM3) to an MQTT broker in real time with remote configuration, OTA updates, and automatic recovery.

**## Overview**
2–3 paragraph description of what the device does, the UM980 GNSS receiver, and what v2.0 adds (provisioning, NTRIP corrections, remote logging). Mention it is designed for unattended outdoor RTK operation.

**## Hardware**
- ESP32 development board
- UM980 GNSS receiver (connected via UART)
- GPIO9 used for SoftAP reset button
- Status LED (GPIO-driven, multi-state blink patterns)

**## Features**
Brief description of each feature (1–2 sentences each):
- GNSS Pipeline — NMEA sentences published to MQTT per sentence type; RTCM3 frames published for external consumers
- Health Heartbeat — JSON published every 60s with uptime, heap, drop counts, NTRIP status, and GNSS fix quality (fix_type, satellites, hdop)
- Remote Logging — All ESP-IDF log output forwarded to MQTT in real time; level configurable at runtime
- NTRIP Corrections — TCP connection to NTRIP caster streams RTCM3 corrections to UM980 for RTK fix
- OTA Firmware Updates — Triggered via MQTT with URL + SHA-256; device downloads, verifies, reboots, and marks slot valid
- Provisioning / SoftAP — Web UI for first-time WiFi and MQTT credential entry; no recompile required
- Command Relay — Forward arbitrary UM980 commands over UART via MQTT publish
- Resilience — Watchdog threads, WiFi/MQTT reconnect timers, automatic reboot on sustained failure

**## First-Time Setup (Provisioning)**
Step-by-step:
1. Flash firmware to ESP32
2. On first boot (no credentials in NVS), device broadcasts `GNSS-Setup` WiFi hotspot (open, no password)
3. Connect to `GNSS-Setup` from any device
4. Open browser — captive portal or navigate to `192.168.4.1` (or the IP shown in serial log)
5. Enter WiFi SSID(s) (up to 3) and password(s)
6. Enter MQTT broker host, port, and optional username/password
7. Click Save — device saves credentials to NVS and reboots into station mode
8. To re-enter SoftAP: hold GPIO9 low for 3 seconds, OR publish `"softap"` to the OTA trigger topic

**## Building and Flashing**
Prerequisites:
- Rust toolchain with xtensa target (`rustup target add xtensa-esp32-espidf` via espup)
- ESP-IDF v5.x (via esp-idf-sys build script)
- `cargo install ldproxy`
- First build: `git submodule update --init --recursive` in ESP-IDF directory

Configuration:
- Copy `src/config.rs.example` to `src/config.rs` (if present) and fill in device-specific values
- Alternatively, use SoftAP provisioning (no config.rs needed for WiFi/MQTT)

Build and flash:
```
cargo build --release
cargo espflash flash --release --monitor
```

**## NTRIP Configuration**
Publish a retained JSON payload to `gnss/{device_id}/ntrip/config`:
```json
{
  "host": "caster.example.com",
  "port": 2101,
  "mountpoint": "MOUNTPOINT",
  "username": "user",
  "password": "pass"
}
```
Device connects immediately and streams RTCM3 corrections to the UM980. Reconnects automatically on connection loss. Settings persisted to NVS — survive reboot.

To disconnect: publish an empty payload to the topic (clears retained config).

**## OTA Firmware Update**
1. Build the new firmware: `cargo build --release`
2. Convert ELF to binary if needed: `esptool.py --chip esp32 elf2image target/xtensa-esp32-espidf/release/esp32-gnssmqtt`
3. Compute SHA-256: `sha256sum esp32-gnssmqtt.bin`
4. Serve the binary over HTTP: `python3 -m http.server 8080`
5. Publish trigger:
   ```
   Topic:   gnss/{device_id}/ota/trigger
   Payload: {"url":"http://192.168.1.X:8080/esp32-gnssmqtt.bin","sha256":"<hex>"}
   Retain:  true
   ```
6. Monitor `/ota/status` and `/log` — device downloads, verifies SHA-256, reboots, and marks slot valid
7. After update: clear the retained trigger (publish empty payload) to prevent re-flash on reconnect

**## Health Heartbeat**
Heartbeat JSON published every 60 seconds to `gnss/{device_id}/heartbeat`:
```json
{
  "uptime_s": 3600,
  "heap_free": 180000,
  "nmea_drops": 0,
  "rtcm_drops": 0,
  "uart_tx_errors": 0,
  "ntrip": "connected",
  "fix_type": 4,
  "satellites": 12,
  "hdop": 0.9
}
```
Field reference:
- `fix_type`: 0=No fix, 1=GPS SPS, 2=DGPS, 4=RTK Fixed, 5=RTK Float, 6=Estimated. `null` if no GGA received.
- `satellites`: Number of satellites in use. `null` if no GGA received.
- `hdop`: Horizontal dilution of precision (lower = better). `null` if no GGA received.
- `ntrip`: `"connected"` or `"disconnected"`

**## Troubleshooting**
Common issues and remedies:
- Device not connecting to WiFi → check NVS credentials; use SoftAP to reconfigure
- MQTT connection fails → verify broker host/port/credentials via SoftAP UI
- OTA fails with HTTP error → check firewall (`sudo ufw allow 8080`); verify IP address
- NTRIP not connecting → verify caster host/port/mountpoint; check `/log` for error details
- Device rebooting repeatedly → check `/log` for watchdog or MQTT disconnect timer; ensure WiFi is stable
