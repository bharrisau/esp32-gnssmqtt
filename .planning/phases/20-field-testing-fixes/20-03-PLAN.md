<objective>
Implement UM980 config persistence to NVS so that when the UM980 GNSS module reboots (power cycle or watchdog), the last known configuration is automatically re-applied without operator intervention.

Purpose: The UM980 loses its config on power cycle or reset. Phase 17-03 added detection of the reboot event (via $devicename banner) but deferred the actual re-apply. This plan implements the deferred feature: save config to NVS blob on every MQTT delivery, read it back on UM980 reboot.

Output: config_relay.rs saves raw config payload to NVS "gnss" namespace on each new config; apply_config() made pub; main.rs UM980 reboot monitor reads NVS and calls apply_config() instead of the current warning-only stub.
</objective>

4. **Add save_gnss_config() function** (private helper, after the spawn function):
   ```rust
   fn save_gnss_config(payload: &[u8], nvs_partition: &EspNvsPartition<NvsDefault>) {
       match EspNvs::new(nvs_partition.clone(), "gnss", true) {
           Ok(mut nvs) => {
               if let Err(e) = nvs.set_blob("gnss_config", payload) {
                   log::warn!("Config relay: NVS save failed: {:?}", e);
               } else {
                   log::info!("Config relay: saved {} bytes to NVS (gnss/gnss_config)", payload.len());
               }
           }
           Err(e) => log::warn!("Config relay: NVS open failed: {:?}", e),
       }
   }
   ```

5. **Make apply_config() pub** — change `fn apply_config(` to `pub fn apply_config(`. This allows main.rs UM980 reboot monitor to call it directly.

2. **Update UM980 reboot monitor thread** to read NVS and call apply_config(). The monitor thread currently has access to `um980_reboot_rx` via move, and `gnss_cmd_tx` via clone. It needs `nvs` clone too.

   The reboot monitor is spawned in a block scope around lines 327-355. Update it to:
   - Move `nvs.clone()` into the thread as `nvs_for_reboot`
   - Move a `gnss_cmd_tx.clone()` into the thread as `gnss_cmd_for_reboot`
   - On `Ok(())` signal, after the 500ms wait, read NVS blob and call apply_config:

   ```rust
   // In the UM980 reboot monitor thread closure, after the existing um980_reboot_tx binding:
   let nvs_for_reboot = nvs.clone();
   let gnss_cmd_for_reboot = gnss_cmd_tx.clone();

   std::thread::Builder::new()
       .stack_size(4096)
       .spawn(move || {
           // ...HWM log unchanged...
           loop {
               match um980_reboot_rx.recv_timeout(crate::config::SLOW_RECV_TIMEOUT) {
                   Ok(()) => {
                       log::warn!("UM980 reboot monitor: reboot detected — waiting 500ms before config re-apply");
                       std::thread::sleep(std::time::Duration::from_millis(500));

                       // Load saved GNSS config from NVS and re-apply to UM980.
                       match esp_idf_svc::nvs::EspNvs::new(nvs_for_reboot.clone(), "gnss", false) {
                           Ok(nvs) => {
                               let mut buf = vec![0u8; 512];
                               match nvs.get_blob("gnss_config", &mut buf) {
                                   Ok(Some(size)) => {
                                       log::info!("UM980 reboot monitor: re-applying {} bytes of saved config", size);
                                       config_relay::apply_config(&buf[..size], &gnss_cmd_for_reboot);
                                       log::info!("UM980 reboot monitor: config re-apply complete");
                                   }
                                   Ok(None) => {
                                       log::info!("UM980 reboot monitor: no saved config in NVS — skipping re-apply");
                                   }
                                   Err(e) => {
                                       log::warn!("UM980 reboot monitor: NVS read error: {:?}", e);
                                   }
                               }
                           }
                           Err(_) => {
                               log::info!("UM980 reboot monitor: 'gnss' NVS namespace not found — no config to re-apply");
                           }
                       }
                   }
                   Err(std::sync::mpsc::RecvTimeoutError::Timeout) => {}
                   Err(std::sync::mpsc::RecvTimeoutError::Disconnected) => {
                       log::error!("UM980 reboot monitor: channel closed — thread parking");
                       loop { std::thread::sleep(std::time::Duration::from_secs(60)); }
                   }
               }
           }
       })
       .expect("UM980 reboot monitor spawn failed");
   ```

   Stack note: the current stack is 4096 bytes. Adding NVS access (which allocates a 512-byte vec) may be tight. Increase stack_size to 8192 for this thread to be safe.
