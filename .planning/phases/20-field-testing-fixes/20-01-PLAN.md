1. Add two compile-time constants before the handler block (or as &[u8] literals inline):
   - `const IOS_SUCCESS_HTML: &[u8] = b"<HTML><HEAD><TITLE>Success</TITLE></HEAD><BODY>Success</BODY></HTML>";`
   - `const WINDOWS_CONNECT_TEST: &[u8] = b"Microsoft Connect Test";`
   - `const WINDOWS_NCSI: &[u8] = b"Microsoft NCSI";`

2. Fix `/hotspot-detect.html` handler: change from `write_all(redirect_html)` to `write_all(IOS_SUCCESS_HTML)`. iOS expects exact `<HTML><HEAD><TITLE>Success</TITLE></HEAD><BODY>Success</BODY></HTML>` with 200 OK — any redirect causes iOS to miss captive detection.

3. Fix `/ncsi.txt` handler: change from `write_all(redirect_html)` to `write_all(WINDOWS_NCSI)`. Older Windows versions probe this URL expecting exactly `Microsoft NCSI`.

4. Add new `/connecttest.txt` handler BEFORE the `/ncsi.txt` handler:
   ```rust
   server.fn_handler("/connecttest.txt", Method::Get, |req| {
       req.into_ok_response()?.write_all(b"Microsoft Connect Test")
   })?;
   ```
   Windows 10/11 probes `http://www.msftconnecttest.com/connecttest.txt` — DNS hijack resolves to 192.168.71.1 — HTTP server on port 80 handles the path. Must return exactly `Microsoft Connect Test` (no trailing newline, no content-type needed).

5. Leave Android handlers unchanged: `/generate_204` (302 redirect), `/connectivitycheck` (302 redirect).
6. Leave `/success.html`, `/library/test/success.html`, `/redirect` handlers unchanged.

Pattern from RESEARCH.md: `req.into_ok_response()?.write_all(b"exact body")` is the correct pattern. The `into_ok_response()` sets Content-Type: text/plain and status 200 automatically.
