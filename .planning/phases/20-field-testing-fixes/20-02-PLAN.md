<task type="auto">
  <name>Task 2: Tune MQTT outbox timeout in sdkconfig.defaults</name>
  <files>sdkconfig.defaults</files>
  <action>
Read sdkconfig.defaults to see current MQTT configuration, then add the outbox timeout config.

In sdkconfig.defaults, add the following MQTT throughput tuning config. Place near any existing CONFIG_MQTT_ entries, or at the end of the file in a new comment block:

```
# MQTT throughput tuning (PERF-1: 5 Hz GNSS support)
# Reduce outbox message expiry from 30s default to 5s.
# At QoS 0, messages not ACKed within 5s (MQTT disconnected) are discarded.
# Prevents outbox memory growth during short MQTT disconnects at high publish rates.
CONFIG_MQTT_OUTBOX_EXPIRED_TIMEOUT_MS=5000
```

This is a conservative change: reduces the window where QoS 0 messages accumulate in the outbox during a transient MQTT disconnect. At QoS 0 (used for NMEA), messages older than 5s are irrelevant anyway (GNSS data is real-time). Default is 30s which would cause ~30s × 40 msg/s = 1200 queued messages during a disconnect, consuming significant heap.

Do NOT change CONFIG_MQTT_TASK_PRIORITY — the research concluded mutex contention is benign at 5 Hz and changing task priority could destabilize other subsystems.
