<objective>
Migrate the NMEA relay, log relay, and heartbeat loop from `Arc<Mutex<EspMqttClient>>` to `SyncSender<MqttMessage>`. Consolidate NMEA to a single topic. Make `LOG_REENTERING` pub. Extend the heartbeat JSON with the new MQTT observability counters.

Purpose: These three are the highest-frequency publishers (NMEA at 40 msg/sec, log relay continuously, heartbeat periodically). Migrating them eliminates mutex contention on the hot path. The RTCM relay and main.rs wiring follow in Plan 03.

Output: Updated `src/nmea_relay.rs`, `src/log_relay.rs`, `src/mqtt.rs`.
</objective>

  <name>Task 1: Migrate nmea_relay.rs to SyncSender and consolidate NMEA topic</name>
  <files>src/nmea_relay.rs</files>
  <action>
Refactor `src/nmea_relay.rs`:

**Signature change:** Replace `client: Arc<Mutex<EspMqttClient<'static>>>` with `mqtt_tx: SyncSender<crate::mqtt_publish::MqttMessage>`. Remove `device_id: String` parameter — the pre-built topic `Arc<str>` is now passed directly instead.

New `spawn_relay` signature:
```rust
pub fn spawn_relay(
    mqtt_tx: std::sync::mpsc::SyncSender<crate::mqtt_publish::MqttMessage>,
    nmea_topic: std::sync::Arc<str>,   // pre-built "gnss/{id}/nmea" — no per-sentence format!()
    nmea_rx: Receiver<(String, String)>,
) -> anyhow::Result<()>
```

**Topic consolidation:** NMEA was previously published to `gnss/{id}/nmea/{sentence_type}` (one topic per sentence type). Change to publish ALL NMEA sentences to the single topic `gnss/{id}/nmea`. The sentence type is visible from the `$GNGGA...` payload prefix — no information is lost. This is a breaking change for subscribers using per-type topics; per CONTEXT.md deferred section, a changelog note may be needed but is out of scope here.

Remove the per-sentence `format!("gnss/{}/nmea/{}", device_id, sentence_type)` call. Use `nmea_topic.clone()` instead (zero allocation on hot path).

**Per-sentence publish:** Replace `client.lock()` + `c.enqueue()` pattern with:
```rust
let payload = raw.into_bytes();
match mqtt_tx.try_send(crate::mqtt_publish::MqttMessage::Nmea {
    topic: nmea_topic.clone(),
    payload,
}) {
    Ok(()) => {}
    Err(std::sync::mpsc::TrySendError::Full(_)) => {
        log::warn!("NMEA relay: publish channel full — sentence dropped");
        crate::gnss::NMEA_DROPS.fetch_add(1, std::sync::atomic::Ordering::Relaxed);
    }
    Err(std::sync::mpsc::TrySendError::Disconnected(_)) => {
        log::error!("NMEA relay: publish channel closed — thread exiting");
        break;
    }
}
```

Keep the `parse_gga_into_atomics` call and the 100-sentence throughput log unchanged.

Remove the `Arc<Mutex<EspMqttClient>>` import and all mutex-related imports. Keep `Receiver`, `RecvTimeoutError`. Add `SyncSender`, `TrySendError`.

**Change 2 — Migrate spawn_log_relay to SyncSender:**
New signature:
```rust
pub fn spawn_log_relay(
    mqtt_tx: std::sync::mpsc::SyncSender<crate::mqtt_publish::MqttMessage>,
    log_topic: std::sync::Arc<str>,   // pre-built "gnss/{id}/log"
```

Inside the relay thread:
- Remove `Arc<Mutex<EspMqttClient>>` and the per-message `client.lock()` + `c.enqueue()` pattern.

- On `TrySendError::Full`: silently drop (preserve LOG-03 behavior — non-blocking, drop when full).
- On `TrySendError::Disconnected`: silently continue (or park — use the same silent continue pattern as `Err(_)` timeout case).

**Critical:** Remove the `LOG_REENTERING.store(true/false, Ordering::Relaxed)` lines from the relay loop. The publish thread now owns the guard. Do not add any log:: calls inside the relay loop (this rule is unchanged from the existing implementation).

Also remove the per-message `format!("gnss/{}/log", device_id)` — the topic is now pre-built and passed in.

Remove `Arc<Mutex<EspMqttClient>>` and `QoS` imports (no longer needed here). Keep `AtomicBool`, `SyncSender`.

The `LOG_TX` channel still holds `String` internally (the log capture path sends to `LOG_TX: OnceLock<SyncSender<String>>`). The relay thread converts `String → Vec<u8>` via `.into_bytes()` before sending to the MqttMessage channel. This is correct and avoids changing the capture path.
