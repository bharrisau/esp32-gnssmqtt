  <action>
    Update main.rs to:

    1. **Add imports** at top of main.rs:
       ```rust
       use tokio::sync::broadcast;
       ```
       (tokio is already a dep; broadcast is part of tokio::sync)

    2. **Create broadcast channel in main()**:
       ```rust
       let (ws_tx, _ws_rx_discard) = broadcast::channel::<String>(16);
       ```
       Capacity 16 is sufficient for ~1 Hz messages. The `_ws_rx_discard` receiver ensures the channel is not considered closed when no WebSocket clients are connected — drop it or assign to `_`.

    3. **Spawn web server task** after the RINEX output dir creation:
       ```rust
       tokio::spawn(web_server::run_web_server(
           config_arc.http_port,
           ws_tx.clone(),
       ));
       ```

    4. **Update run_decode_task call** to pass ws_tx:
       ```rust
       tokio::spawn(run_decode_task(
           msg_rx,
           config_arc.output_dir.clone(),
           config_arc.device_id.clone(),
           ws_tx,
       ));
       ```

    5. **Update run_decode_task signature and body**:
       Add `ws_tx: broadcast::Sender<String>` as fourth parameter.

       Add `let mut gsv_acc = nmea_parse::GsvAccumulator::new();` before the while loop.

       Change the match inside the while loop from `if let mqtt::MqttMessage::Rtcm(payload) = msg` to a full match:
       ```rust
       match msg {
           mqtt::MqttMessage::Rtcm(payload) => {
               // existing RTCM decode code unchanged
               let events = rtcm_decode::decode_rtcm_payload(&payload, &mut epoch_buf);
               for event in events {
                   match event {
                       observation::RtcmEvent::Epoch(group) => { /* unchanged */ }
                       observation::RtcmEvent::Ephemeris(eph) => { /* unchanged */ }
                   }
               }
           }
           mqtt::MqttMessage::Nmea(payload) => {
               if let Ok(s) = std::str::from_utf8(&payload) {
                   if let Some(state) = gsv_acc.feed(s) {
                       if let Ok(json) = serde_json::to_string(&state) {
                           let _ = ws_tx.send(json); // lagged receivers dropped silently
                       }
                   }
               }
           }
           mqtt::MqttMessage::Heartbeat(payload) => {
               if let Some(tagged) = nmea_parse::tag_heartbeat(&payload) {
                   let _ = ws_tx.send(tagged);
               }
           }
       }
       ```

    6. **Add serde_json to run_decode_task** — serde_json is now a gnss-server dep (added in 25-01). No additional import needed beyond `use serde_json;` if not already imported, or just call `serde_json::to_string` directly.

    7. **Ensure mod declarations** are present at top of main.rs:
       ```rust
       mod nmea_parse;
       mod web_server;
       ```
       (These may have been added as stubs in 25-01; ensure they are present and correct.)

    After these changes, `run_web_server` logs "Listening on 0.0.0.0:{port}" at startup. The server binary now starts both the MQTT supervisor and the HTTP/WebSocket server.
