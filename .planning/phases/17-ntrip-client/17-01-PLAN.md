Module contents:

**1. Imports and NTRIP_STATE atomic:**
```rust
use esp_idf_svc::hal::uart::UartDriver;
use esp_idf_svc::nvs::{EspNvs, EspNvsPartition, NvsDefault};
use std::io::{Read, Write as IoWrite};
use std::net::TcpStream;
use std::sync::Arc;
use std::sync::atomic::{AtomicU8, Ordering};
use std::sync::mpsc::{Receiver, RecvTimeoutError};
use std::time::Duration;

pub static NTRIP_STATE: AtomicU8 = AtomicU8::new(0); // 0=disconnected, 1=connected
```

**2. NtripConfig struct with Default:**
```rust
#[derive(Default, Clone)]
pub struct NtripConfig {
    pub host:       String,
    pub port:       u16,      // default 2101
    pub mountpoint: String,
    pub user:       String,
    pub pass:       String,
}
impl NtripConfig {
    pub fn is_valid(&self) -> bool { !self.host.is_empty() && !self.mountpoint.is_empty() }
}
```
Default port: 2101 — implement in Default derive via `..Default::default()` pattern or a custom Default impl that sets port = 2101.

**3. NVS keys (all ≤ 15 chars — verify each):**
- `"ntrip_host"` (10), `"ntrip_port_hi"` (13), `"ntrip_port_lo"` (13), `"ntrip_mount"` (11), `"ntrip_user"` (10), `"ntrip_pass"` (10)
NVS namespace: `"ntrip"` — use `EspNvs::new(nvs, "ntrip", true)`.

**4. load_ntrip_config(nvs) and save_ntrip_config(config, nvs) functions:**
- Load: get_str for host/mountpoint/user/pass, two-u8 get_u8 for port (ntrip_port_hi/ntrip_port_lo), default port 2101 if keys absent.
- Save: set_str for strings, set_u8 for port bytes (high byte = port >> 8, low byte = port & 0xFF).
- Return `NtripConfig` (not Result) from load — use unwrap_or_default on individual field errors.

**5. base64_encode(input: &str) -> String:**
Implement the RFC 4648 §4 standard base64 alphabet encoder from RESEARCH.md (no external crate). Encode `"user:pass"` for Authorization header.

**6. build_ntrip_request(config: &NtripConfig) -> String:**
```
GET /{mountpoint} HTTP/1.0\r\n
User-Agent: NTRIP esp32-gnssmqtt/1.0\r\n
Accept: */*\r\n
[Authorization: Basic {base64}\r\n]  -- only if user non-empty
\r\n
```

**7. read_ntrip_headers(stream: &mut TcpStream) -> Result<bool, std::io::Error>:**
Read byte-by-byte into a 512-byte header buffer until `\r\n\r\n` is seen. Return `Ok(true)` if first line starts with `"ICY 200 OK"`. Return `Ok(false)` for other responses (log them). Return `Err` on I/O error or EOF before headers end.

**8. run_ntrip_session(config, uart, config_rx) -> Result<NtripConfig, std::io::Error>:**
- Connect: `TcpStream::connect(format!("{}:{}", config.host, config.port))?`
- Set timeouts: `stream.set_read_timeout(Some(Duration::from_secs(60)))?` and `stream.set_write_timeout(Some(Duration::from_secs(10)))?`
- Send request: `stream.write_all(build_ntrip_request(config).as_bytes())?`
- Read headers: validate ICY 200 OK; return Err on bad response
- Set `NTRIP_STATE.store(1, Ordering::Relaxed)`
- Log HWM at session start (project standard)
- Streaming loop: `stream.read(&mut buf)?` with 512-byte stack buf
  - 0 bytes → caster closed → break
  - n bytes → `uart.write(&buf[..n])` (log warn on error but continue)
  - During backoff check_config_rx: use `try_recv()` — if Ok(payload) → parse and return Ok(new_config) to trigger reconnect with new config
- On loop exit: `NTRIP_STATE.store(0, Ordering::Relaxed)`, return Ok(config) for clean exit

**9. spawn_ntrip_client(uart, ntrip_config_rx, nvs):**
```rust
pub fn spawn_ntrip_client(
    uart: Arc<UartDriver<'static>>,
    ntrip_config_rx: Receiver<Vec<u8>>,
    nvs: EspNvsPartition<NvsDefault>,
) -> anyhow::Result<()>
```
Thread body:
- Stack size: 8192
- Log HWM at entry
- Load config from NVS on startup
- Main loop:
  - If config not valid: `recv_timeout(SLOW_RECV_TIMEOUT)` → parse JSON → save → update config → continue
  - If config valid: call `run_ntrip_session`; on error → log, apply backoff via `ntrip_config_rx.recv_timeout(backoff_delay)`:
    - `Ok(payload)` → parse JSON → save → update config → reset backoff
    - `Err(Timeout)` → retry with incremented backoff (steps: 5s, 10s, 20s, 40s cap)
    - `Err(Disconnected)` → break

**JSON config parsing (parse_ntrip_config_payload(data: &[u8]) -> Option<NtripConfig>):**
Follow existing config_relay.rs / provisioning.rs JSON parsing pattern. The JSON payload is:
`{"host":"...","port":2101,"mountpoint":"...","user":"...","pass":"..."}`
Use manual string extraction (no serde — project has none). Extract fields using the `extract_json_str` / `extract_json_number` pattern from config_relay.rs or equivalent. Port is numeric; default to 2101 if absent.

KNOWN RACE documented in code comment: UartDriver::write(&self) not mutex-protected; NTRIP thread and GNSS TX thread can write concurrently. Commands are rare; risk is low. Document with `// KNOWN-RACE: see RESEARCH.md Pitfall 3`.
