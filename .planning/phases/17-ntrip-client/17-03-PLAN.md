**log_relay.rs Change B — ANSI strip on C vprintf path:**
Add a private function `strip_ansi(s: String) -> String` that removes ANSI SGR escape sequences. These arrive as `\x1b[` followed by digits/semicolons followed by `m`. The simpler form `[0;32m` (without the leading `\x1b`) also appears from some ESP-IDF loggers.

Strip both patterns:
- Full: `\x1b[` followed by `[0-9;]*` followed by `m`
- Bare: `[` followed by `[0-9;]+` followed by `m` — only strip this form if it appears at the start of the string or after a stripped full escape (to avoid stripping valid bracket characters in message text)

Practical approach: strip `\x1b[[0-9;]*m` globally first (covers the standard form), then strip `\x1b\[[0-9;]*m` if the previous missed any (same pattern, just ensure both bracket forms are caught). For the bare-bracket form `[0;32m` that ESP-IDF sometimes emits: strip `\x1b[0-9;]*m` (ESC is byte 0x1b = 27).

The implementation should NOT use regex (no regex crate in this project). Implement as a simple in-place scan:

```rust
fn strip_ansi(s: String) -> String {
    // Strip ANSI SGR escape sequences: ESC [ ... m
    // ESC = 0x1B. Matches sequences like \x1b[0;32m, \x1b[0m, \x1b[1;33m.
    // C component logs (via vprintf hook) arrive with these codes intact.
    let bytes = s.as_bytes();
    let mut out = Vec::with_capacity(bytes.len());
    let mut i = 0;
    while i < bytes.len() {
        if bytes[i] == 0x1B && i + 1 < bytes.len() && bytes[i + 1] == b'[' {
            // Skip ESC [ ... m sequence
            i += 2;
            while i < bytes.len() && (bytes[i].is_ascii_digit() || bytes[i] == b';') {
                i += 1;
            }
            if i < bytes.len() && bytes[i] == b'm' {
                i += 1; // consume 'm'
            }
        } else {
            out.push(bytes[i]);
            i += 1;
        }
    }
    String::from_utf8_lossy(&out).into_owned()
}
```

In `rust_log_try_send`, apply the strip before try_send:
```rust
let s = unsafe { std::ffi::CStr::from_ptr(msg) }
    .to_string_lossy()
    .into_owned();
let s = strip_ansi(s);   // remove ANSI color codes from C-path log output
let _ = tx.try_send(s);
```

Note: the MqttLogger Rust path (log:: calls) does NOT need stripping — EspLogger formats Rust log records without ANSI codes in the path that reaches MqttLogger::log(). Only the C vprintf path produces ANSI codes.

**main.rs Change B — Spawn UM980 reboot monitor thread:**
After the config_relay spawn (Step 10 area), add a new thread that waits for reboot signals and re-triggers the config relay. The config relay re-apply needs access to `gnss_cmd_tx` and NVS (to reload the stored UM980 config commands).

```rust
// UM980 reboot monitor: re-applies startup configuration when UM980 resets.
// Listens on um980_reboot_rx; on signal, calls the same config re-apply
// path used at startup (config_relay::apply_stored_config or equivalent).
{
    let gnss_cmd_tx_reboot = gnss_cmd_tx.clone();
    let nvs_reboot = nvs.clone();
    std::thread::Builder::new()
        .stack_size(4096)
        .spawn(move || {
            let hwm_words = unsafe {
                esp_idf_svc::sys::uxTaskGetStackHighWaterMark(core::ptr::null_mut())
            };
            log::info!("[HWM] {}: {} words ({} bytes) stack remaining at entry",
                "UM980 reboot monitor", hwm_words, hwm_words * 4);
            loop {
                match um980_reboot_rx.recv_timeout(crate::config::SLOW_RECV_TIMEOUT) {
                    Ok(()) => {
                        log::warn!("UM980 reboot monitor: re-applying startup configuration");
                        // Re-apply using the same function as startup config relay.
                        // Wait 500ms first: UM980 needs time to initialise its command parser
                        // after the reboot banner before it can accept configuration commands.
                        std::thread::sleep(std::time::Duration::from_millis(500));
                        if let Err(e) = config_relay::apply_stored_config(&gnss_cmd_tx_reboot, &nvs_reboot) {
                            log::error!("UM980 reboot monitor: config re-apply failed: {:?}", e);
                        }
                    }
                    Err(std::sync::mpsc::RecvTimeoutError::Timeout) => {}
                    Err(std::sync::mpsc::RecvTimeoutError::Disconnected) => {
                        log::error!("UM980 reboot monitor: channel closed — thread parking");
                        loop { std::thread::sleep(std::time::Duration::from_secs(60)); }
                    }
                }
            }
        })
        .expect("UM980 reboot monitor spawn failed");
}
log::info!("UM980 reboot monitor started");
```
