- "wifi_pass_2"  (11) — str: password for network 2

- "mqtt_port_hi" (12) — u8: high byte of MQTT port

```rust
//! Provisioning module — SoftAP web portal and NVS credential storage.
//!
//! Boot-path logic: main.rs reads NVS before WiFi init; if no credentials (or force_softap set),
//! calls run_softap_portal() instead of wifi_connect().
```

**Function 1: `has_wifi_credentials`**
- Opens NVS namespace "prov" read-only (false). Returns false if namespace does not yet exist (Err → false).
- Reads "wifi_count" u8. Returns true if count > 0.
- Signature: `pub fn has_wifi_credentials(nvs_partition: &EspNvsPartition<NvsDefault>) -> bool`
- Uses `nvs_partition.clone()` to open NVS without consuming the partition (partition is Clone).

**Function 2: `check_and_clear_force_softap`**
- Opens NVS namespace "prov" read-write (true). Returns false on error.
- Reads "force_softap" u8. If Some(1), writes 0 back and returns true. Otherwise returns false.
- Signature: `pub fn check_and_clear_force_softap(nvs_partition: &EspNvsPartition<NvsDefault>) -> bool`

**Function 3: `load_wifi_networks`**
- Opens NVS namespace "prov" read-only. On error returns empty Vec.
- Reads "wifi_count" u8. For each index 0..count.min(3): reads "wifi_ssid_{i}" and "wifi_pass_{i}" with a 65-byte stack buffer each.
- Returns `Vec<(String, String)>` of (ssid, pass) pairs (only pairs where both keys are present).
- Signature: `pub fn load_wifi_networks(nvs_partition: &EspNvsPartition<NvsDefault>) -> Vec<(String, String)>`

**Function 4: `load_mqtt_config`**
- Opens NVS namespace "prov" read-only. On error returns None.
- Reads "mqtt_host" (65-byte buf), "mqtt_port_hi" u8, "mqtt_port_lo" u8, "mqtt_user" (65-byte buf), "mqtt_pass" (65-byte buf).
- Reconstructs port as `(hi as u16) << 8 | (lo as u16)`. If port is 0, use 1883 as default.
- Returns `Some((host, port, user, pass))` only if mqtt_host is non-empty. Otherwise None.
- Signature: `pub fn load_mqtt_config(nvs_partition: &EspNvsPartition<NvsDefault>) -> Option<(String, u16, String, String)>`

**Function 5: `set_force_softap`**
- Opens NVS namespace "prov" read-write. Writes "force_softap" = 1. Logs warning on error.
- Used by GPIO9 monitor and MQTT "softap" trigger (Plan 15-03).
- Signature: `pub fn set_force_softap(nvs_partition: &EspNvsPartition<NvsDefault>)`

**Imports needed:**

**Implementation steps:**

1. Start SoftAP:


2. Start HTTP server with stack_size: 10240:
```rust

use embedded_svc::io::{Read, Write};

```

3. Register GET "/" handler returning `PROV_HTML` (see below).

4. Register POST "/save" handler. The handler must:
   - Check `content_len()`. If > 2048, return 413.
   - Read body into a `vec![0u8; len]` buffer using `connection.read(&mut buf)`.
   - Call `parse_form_field` (private helper, see below) to extract ssid0/pass0, ssid1/pass1, ssid2/pass2, mqtt_host, mqtt_port, mqtt_user, mqtt_pass.
   - Build a `Vec<(&str, &str)>` of networks where ssid is non-empty (up to 3).
   - Call `save_credentials(nvs_partition.clone(), &networks, mqtt_host, port, mqtt_user, mqtt_pass)`.
   - Write response: `"Saved. Rebooting in 1 second..."`.
   - Spawn a thread that sleeps 1 second then calls `unsafe { esp_idf_svc::sys::esp_restart(); }`.

   The `nvs_partition` must be moved into the POST handler closure. Use `Arc<Mutex<EspNvsPartition<NvsDefault>>>` or simply clone it per the Clone guarantee. Since only one POST submit is expected, clone once before registering the handler and move the clone into the closure.

5. Add `save_credentials` private function:
```rust
fn save_credentials(
    nvs_partition: EspNvsPartition<NvsDefault>,
    networks: &[(&str, &str)],
    mqtt_host: &str,
    mqtt_port: u16,
    mqtt_user: &str,
    mqtt_pass: &str,
) -> anyhow::Result<()> {
    let mut nvs = EspNvs::new(nvs_partition, "prov", true)?;
    let count = networks.len().min(3) as u8;
    nvs.set_u8("wifi_count", count)?;
    for (i, (ssid, pass)) in networks.iter().enumerate().take(3) {
        nvs.set_str(&format!("wifi_ssid_{}", i), ssid)?;
        nvs.set_str(&format!("wifi_pass_{}", i), pass)?;
    }
    nvs.set_str("mqtt_host", mqtt_host)?;
    nvs.set_u8("mqtt_port_hi", (mqtt_port >> 8) as u8)?;
    nvs.set_u8("mqtt_port_lo", (mqtt_port & 0xFF) as u8)?;
    nvs.set_str("mqtt_user", mqtt_user)?;
    nvs.set_str("mqtt_pass", mqtt_pass)?;
    Ok(())
}
```

6. Add `parse_form_field` private helper:
```rust
fn parse_form_field<'a>(body: &'a str, key: &str) -> Option<&'a str> {
    let search = format!("{}=", key);
    let start = body.find(&search)? + search.len();
    let end = body[start..].find('&').map(|i| start + i).unwrap_or(body.len());
    Some(&body[start..end])
}
```
Note: does not handle percent-encoding. Document in a comment: "Known limitation: passwords containing &, %, +, =, or non-ASCII characters are not supported in v2.0."

7. 300-second no-client timeout loop (runs after HTTP server is started, keeping server handle alive):
```rust
fn count_softap_clients() -> u8 {
    let mut sta_list = esp_idf_svc::sys::wifi_sta_list_t::default();
    let ret = unsafe { esp_idf_svc::sys::esp_wifi_ap_get_sta_list(&mut sta_list as *mut _) };
    if ret == 0 { sta_list.num as u8 } else { 0 }
}
```
After server starts, loop every 1 second: if client count > 0 reset timer; if 300 continuous seconds with no client, log and `esp_restart()` without setting force_softap (so next boot tries STA mode with stored credentials).

The `run_softap_portal` function keeps `server` alive in scope until the 300s timeout restarts. It never returns normally — either credentials are submitted (restart from handler thread) or 300s elapses (restart from timeout loop).
