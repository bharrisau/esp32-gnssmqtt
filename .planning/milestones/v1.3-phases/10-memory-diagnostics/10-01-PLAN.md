<objective>
Add a FreeRTOS stack high-water mark (HWM) log line at the entry point of every spawned thread so stack headroom is visible in the startup log.

Purpose: Stack pressure is currently invisible — threads can silently overflow their configured stack sizes. HWM logging at thread entry gives operators a sanity check that each thread's configured stack is adequate (values below ~500 bytes remaining at entry would be concerning).

Output: Every `move ||` closure passed to `thread::Builder::new().spawn()` logs `[HWM] <thread-name>: N words (N bytes) stack remaining` as its first statement. No new dependencies. No behavioral changes — purely additive logging.
</objective>
