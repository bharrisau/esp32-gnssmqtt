# Plan 09-01: Channel Bounding + UART TX Error Logging

**Phase**: 9 — Channel + Loop Hardening
**Requirements**: HARD-01, HARD-02
**Depends on**: Phase 8 complete (OTA wired)

## Goal

Convert all four unbounded `mpsc::channel()` channels to bounded `sync_channel()` with documented
capacities. Log (and count) UART TX write failures instead of silently discarding them.

## Problem Statement

Four channels in the current codebase use unbounded `mpsc::channel()`:
- `subscribe_tx` (main.rs:130) — pump → subscriber, signals Connected events
- `config_tx` (main.rs:133) — pump → config_relay, forwards `/config` payloads
- `ota_tx` (main.rs:136) — pump → OTA task, forwards `/ota/trigger` payloads
- `cmd_tx` (gnss.rs:114) — config_relay/bridge → gnss TX thread

Unbounded channels grow without limit if the consumer falls behind, risking OOM on an embedded
device with limited heap. The MQTT pump thread also calls blocking `send()` on `config_tx` and
`ota_tx` — if consumer is slow, this can delay `connection.next()` calls, causing MQTT heartbeat
timeouts.

Additionally, UART TX write failures in the gnss TX thread are silently discarded:
```
let _ = uart_tx.write(line.as_bytes());
let _ = uart_tx.write(b"\r\n");
```
This hides UM980 command delivery failures that operators cannot see.

## Changes Required

### 1. `src/gnss.rs`

**Channel**: Change `cmd_tx/cmd_rx` from unbounded to bounded:
```rust
// Before:
let (cmd_tx, cmd_rx) = mpsc::channel::<String>();

// After:
// Bounded to 16: a config batch is typically ≤ 16 commands (with 100ms delay between sends).
// Capacity 16 prevents config_relay from blocking UART TX drain on large batches.
let (cmd_tx, cmd_rx) = mpsc::sync_channel::<String>(16);
```

**Return type**: `spawn_gnss` currently returns `Sender<String>`. Change to `SyncSender<String>`.
Update the function signature and doc comment.

**UART TX error logging**: Replace silent `let _ = ...` discards with logged error handling.
Add a module-level atomic error counter:
```rust
use std::sync::atomic::{AtomicU32, Ordering};
static UART_TX_ERRORS: AtomicU32 = AtomicU32::new(0);
```

TX thread loop body becomes:
```rust
for line in cmd_rx.iter() {  // iter() used here; recv_timeout conversion is Plan 09-02
    if let Err(e) = uart_tx.write(line.as_bytes()) {
        let n = UART_TX_ERRORS.fetch_add(1, Ordering::Relaxed) + 1;
        log::warn!("GNSS TX: write error #{}: {:?}", n, e);
    }
    if let Err(e) = uart_tx.write(b"\r\n") {
        let n = UART_TX_ERRORS.fetch_add(1, Ordering::Relaxed) + 1;
        log::warn!("GNSS TX: CRLF write error #{}: {:?}", n, e);
    }
}
```

Note: `cmd_rx.iter()` is used here to preserve the `for x in receiver` pattern; Plan 09-02
converts this to `recv_timeout()`.

**Imports**: Add `SyncSender` and `AtomicU32` to the use list:
```rust
use std::sync::mpsc::{self, Receiver, SyncSender, TrySendError};
use std::sync::atomic::{AtomicU32, Ordering};
```

### 2. `src/config_relay.rs`

`spawn_config_relay` receives a `Sender<String>` for `gnss_cmd_tx`. Change to `SyncSender<String>`.
The `gnss_cmd_tx.send()` calls in `apply_config` may block briefly if channel is full — this is
acceptable since config_relay is not a hot-path thread (infrequent, operator-triggered).

```rust
// Before:
pub fn spawn_config_relay(gnss_cmd_tx: Sender<String>, ...) -> ...

// After:
pub fn spawn_config_relay(gnss_cmd_tx: SyncSender<String>, ...) -> ...
```

Update imports: `use std::sync::mpsc::{Receiver, SyncSender};`

### 3. `src/uart_bridge.rs`

`spawn_bridge` receives a `Sender<String>`. Change to `SyncSender<String>`.

The user-interactive `cmd_tx.send(s)` should NOT block if the channel is full (user typed more
than 16 commands without them being drained). Change to `try_send` with warn:

```rust
// Before:
cmd_tx.send(s).unwrap_or_else(|e| log::warn!("UART bridge: GNSS cmd channel closed: {:?}", e));

// After:
match cmd_tx.try_send(s) {
    Ok(_) => {}
    Err(std::sync::mpsc::TrySendError::Full(_)) => {
        log::warn!("UART bridge: GNSS cmd channel full — command dropped");
    }
    Err(std::sync::mpsc::TrySendError::Disconnected(_)) => {
        log::warn!("UART bridge: GNSS cmd channel closed");
    }
}
```

Update imports: `use std::sync::mpsc::SyncSender;`

### 4. `src/main.rs`

Convert three unbounded channels to bounded `sync_channel`:

```rust
// Step 9: subscribe signal — pump → subscriber
// Bounded to 2: at most one Connected event queued while subscriber processes the previous one.
// A second Connected event (broker restart) while subscriber is busy will queue; beyond 2 is impossible.
let (subscribe_tx, subscribe_rx) = std::sync::mpsc::sync_channel::<()>(2);

// Step 9b: config payload — pump → config_relay
// Bounded to 4: config is operator-triggered (rare). 4 covers a retained message on reconnect
// plus a small burst. pump uses try_send() so it never blocks connection.next().
let (config_tx, config_rx) = std::sync::mpsc::sync_channel::<Vec<u8>>(4);

// Step 9c: OTA trigger — pump → OTA task
// Bounded to 1: at most one OTA operation can be queued. A second trigger while OTA is running
// is dropped (pump uses try_send). Prevents double-flash from re-delivered retained triggers.
let (ota_tx, ota_rx) = std::sync::mpsc::sync_channel::<Vec<u8>>(1);
```

No type annotation changes needed on local variables — Rust infers `SyncSender<T>` from
`sync_channel` return.

### 5. `src/mqtt.rs`

`pump_mqtt_events` currently takes `Sender<()>`, `Sender<Vec<u8>>`, `Sender<Vec<u8>>`.
Change to `SyncSender` variants. Change all `send()` calls to `try_send()` with appropriate
error handling.

**Signature**:
```rust
pub fn pump_mqtt_events(
    mut connection: EspMqttConnection,
    subscribe_tx: SyncSender<()>,
    config_tx: SyncSender<Vec<u8>>,
    ota_tx: SyncSender<Vec<u8>>,
    led_state: Arc<AtomicU8>,
) -> !
```

**`subscribe_tx` send on Connected**:
```rust
// Try to signal subscriber; if channel is full, subscriber is already queued to re-subscribe
// on the previous Connected event — this signal can be dropped safely.
match subscribe_tx.try_send(()) {
    Ok(_) => {}
    Err(TrySendError::Full(_)) => {
        log::warn!("pump: subscribe signal channel full — subscriber already queued");
    }
    Err(TrySendError::Disconnected(_)) => {
        log::error!("pump: subscribe channel closed");
    }
}
```

**`config_tx` send on Received `/config`**:
```rust
match config_tx.try_send(data.to_vec()) {
    Ok(_) => {}
    Err(TrySendError::Full(_)) => log::warn!("pump: config channel full — payload dropped"),
    Err(TrySendError::Disconnected(_)) => log::warn!("pump: config channel closed"),
}
```

**`ota_tx` send on Received `/ota/trigger`**:
```rust
match ota_tx.try_send(data.to_vec()) {
    Ok(_) => log::info!("OTA trigger received, payload len={}", data.len()),
    Err(TrySendError::Full(_)) => log::warn!("pump: OTA channel full — trigger dropped (OTA in progress?)"),
    Err(TrySendError::Disconnected(_)) => log::warn!("pump: OTA channel closed"),
}
```

**Imports**: Add `SyncSender` to the mpsc use:
```rust
use std::sync::mpsc::{Receiver, SyncSender};
```

## Notes

- `config_relay.rs` `apply_config()` continues to use blocking `send()` into `cmd_tx` — this is
  correct because config_relay is not a hot-path thread and blocking on a full channel (16 slots)
  is acceptable during large config batches.
- `recv_timeout()` conversion for all channel receivers is deferred to Plan 09-02 (HARD-06).
- The `UART_TX_ERRORS` counter will be read in Phase 13 (health telemetry) for the status payload.
