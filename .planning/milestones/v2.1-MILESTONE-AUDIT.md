## Tech Debt by Phase

### Phase 24 — RINEX Files
- **RINEX-04**: `rnx2rtkp` acceptance test against live device FFFEB5 not run — requires real RTCM3 stream. Code is structurally correct per unit tests (D19.12 formatter, RINEX 2.11 headers). Deferred to hardware testing checklist.
- **Hourly rotation**: Requires real UTC hour boundary; cannot be verified in unit tests. Deferred to hardware testing.

### Phase 25 — Web UI
- **GN-talker gap (UI-02/UI-03)**: nmea 0.7 crate ignores `$GN` (combined-constellation) talker. UM980 emits `$GN` sentences by default. Skyplot and SNR chart will not update with real device data. The `GsvAccumulator.feed()` gracefully returns `None` rather than erroring. Fix: either configure UM980 to emit per-constellation talkers (`$GP`, `$GL`, `$GA`, `$GB`), or upgrade to an nmea crate that supports `$GN` talker parsing.

### Cross-Milestone
- **gnss-nvs firmware integration**: Firmware still calls `EspNvs` directly. `NvsStore` trait exists but is not used in firmware yet. Deferred per NOSTD-F02.
- **Nyquist compliance**: All 4 v2.1 phases have VALIDATION.md with `nyquist_compliant: false`. Planning artifacts were created but not updated post-execution. Run `/gsd:validate-phase N` for each to retroactively mark compliant.
