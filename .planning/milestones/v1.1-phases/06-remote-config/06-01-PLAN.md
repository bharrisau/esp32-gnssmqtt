**spawn_config_relay** — public entry point, called from main.rs:
```rust
pub fn spawn_config_relay(
    gnss_cmd_tx: std::sync::mpsc::Sender<String>,
    config_rx: std::sync::mpsc::Receiver<Vec<u8>>,
) -> anyhow::Result<()>
```
- Use `std::thread::Builder::new().stack_size(8192).spawn(...)` (same as all other threads).
- The thread loop: `for payload in &config_rx { ... }` — blocks until pump sends a payload.
- Guard: if `payload.is_empty()`, log `"Config relay: empty payload — retained message cleared, skipping"` and `continue`.
- Compute `let hash = djb2_hash(&payload)`. Compare to `let mut last_hash: u32 = 0` (initialized once, persists across iterations).
- If `hash == last_hash`: log `"Config relay: payload unchanged (hash {:#010x}), skipping"` and `continue`.
- Else: update `last_hash = hash`, log `"Config relay: new config payload, hash {:#010x}"`, then call `apply_config(&payload, &gnss_cmd_tx)`.
- After the for-loop (channel closed): log `"Config relay: channel closed — thread exiting"` then loop forever sleeping 60s (diverging thread pattern).

**djb2_hash** — private helper:
```rust
fn djb2_hash(data: &[u8]) -> u32 {
    let mut hash: u32 = 5381;
    for &byte in data {
        hash = hash.wrapping_mul(33).wrapping_add(byte as u32);
    }
    hash
}
```

**apply_config** — private helper dispatching to parse path:
- Convert payload to `&str` via `std::str::from_utf8(payload)`. If Err: log error and return.
- If `text.trim_start().starts_with('{')`: call `parse_config_json(text)`. If None: log `"Config relay: JSON parse failed, discarding payload"` and return. If Some((delay_ms, commands)): iterate.
- Else (plain text fallback): `let commands: Vec<&str> = text.lines().filter(|l| !l.is_empty()).collect(); let delay_ms: u64 = 100;`
- Command loop: for each `cmd` in `commands`, log `"Config relay: sending command: {:?}"`. Call `gnss_cmd_tx.send(cmd.to_string())`. If `Err(e)`: log `"Config relay: gnss_cmd_tx send failed (TX thread dead?): {:?}"` and `return` (abandon remaining commands). If Ok: `std::thread::sleep(std::time::Duration::from_millis(delay_ms))`.
- Note: commands must NOT include `\r\n` — the gnss.rs TX thread appends `\r\n` to every Sender<String> message.

**parse_config_json** — private helper, returns `Option<(u64, Vec<&str>)>`:
- Input: JSON string with fixed schema `{"delay_ms": N, "commands": ["CMD1", "CMD2"]}`.
- Extract `delay_ms`: find `"delay_ms"` in text, find `:` after it, parse the digits that follow. Default 100 if field absent or parse fails.
- Extract commands: find `[` and `]`, extract content between them, split on `,`, strip leading/trailing whitespace and surrounding `"` from each item via `strip_prefix('"')?.strip_suffix('"')?`.
- Return `None` if commands array is empty or `]` not found after `[`.
- Return `Some((delay_ms, commands))` otherwise.
- Limitation to document in a comment: handles the specified schema only; does not handle escaped quotes inside command strings (UM980 commands contain no special characters).
